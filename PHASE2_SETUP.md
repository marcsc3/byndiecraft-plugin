# Phase 2: AI-Powered Ticket Implementation - Setup Guide

Complete guide to set up and test the AI ticket implementation system.

## Architecture Overview

```
┌──────────────────┐
│  Minecraft       │
│  Player throws   │
│  book into       │
│  AI Hopper       │
└────────┬─────────┘
         │
         ↓ HTTP POST
┌────────────────────┐
│  MCP Server        │
│  (Node.js)         │
│  localhost:3000    │
└────┬───────────┬───┘
     │           │
     ↓           ↓
┌─────────┐ ┌──────────┐
│ Bedrock │ │ GitHub   │
│ Claude  │ │ gh CLI   │
└─────────┘ └──────────┘
```

## Prerequisites

- ✅ Java 25 (for Minecraft server)
- ✅ Node.js 18+ (for MCP server)
- ✅ GitHub CLI (`gh`)
- ✅ AWS credentials configured (profile: `development`)
- ✅ Paper/Spigot 1.21 Minecraft server

## Step 1: Setup MCP Server

### 1.1 Install Dependencies

```bash
cd mcp-server
npm install
```

### 1.2 Authenticate with GitHub

```bash
gh auth login
```

Follow the prompts. This only needs to be done once.

### 1.3 Verify AWS Credentials

```bash
aws configure list --profile development
```

You should see your credentials for the `development` profile.

### 1.4 Start the Server

```bash
npm start
```

You should see:
```
================================================================================
🎮 Byndiecraft MCP Server
================================================================================
Server running on http://localhost:3000
AWS Region: eu-central-1
AWS Profile: development
Model: arn:aws:bedrock:eu-central-1:745091492598:application-inference-profile/ltk3rb6ikzf9
Default Repo: bynder/byndiecraft-plugin
================================================================================
Ready to implement tickets! 🚀
```

### 1.5 Test the Server

Open a new terminal:

```bash
curl http://localhost:3000/health
```

Expected response:
```json
{"status":"ok","timestamp":"2026-06-10T10:00:00.000Z"}
```

## Step 2: Setup Minecraft Plugin

### 2.1 Build the Plugin

```bash
mvn clean package
```

### 2.2 Install the Plugin

```bash
cp target/byndiecraft-plugin-1.0.0-SNAPSHOT.jar /path/to/minecraft/plugins/
```

### 2.3 Configure the Plugin

Edit `plugins/Byndiecraft/config.yml`:

```yaml
ai:
  enabled: true
  mcp_endpoint: "http://localhost:3000/implement-ticket"
  
  bedrock:
    aws_profile: "development"
    region: "eu-central-1"
    model_id: "arn:aws:bedrock:eu-central-1:745091492598:application-inference-profile/ltk3rb6ikzf9"
```

### 2.4 Restart Minecraft Server

```bash
# Restart your server
```

Check the logs for:
```
[Byndiecraft] ✓ AI system initialized (Phase 2)
[Byndiecraft] MCP endpoint: http://localhost:3000/implement-ticket
```

## Step 3: Test End-to-End

### 3.1 Setup AI Hopper in Minecraft

1. **Join the Minecraft server**
2. **Place a hopper** somewhere
3. **Look at the hopper** and run:
   ```
   /jiraboard setaihopper
   ```
4. You should see:
   ```
   ✓ AI Hopper set at: (X, Y, Z)
   Throw written books with Jira ticket keys into this hopper to trigger AI implementation!
   ```

### 3.2 Create a Test Book

1. **Get a written book**:
   ```
   /give @s written_book
   ```

2. **Write the book** with a Jira ticket key as the title:
   - Title: `TEST-123` (or any valid Jira ticket)
   - Content: (optional)

3. **Sign the book**

### 3.3 Throw the Book into the Hopper

1. **Throw the book** into the AI hopper (Q key or drop)
2. **Watch for**:
   - ✨ Portal particles above the hopper
   - 🔮 Enchantment table sound
   - Chat message: "🤖 Claude is implementing TEST-123..."
   - Chat message: "The hopper has consumed your book!"

### 3.4 Wait for the Result

After 10-30 seconds, you should see:

**Success:**
- ✅ Chat message: "PR Created: https://github.com/..."
- 📖 Book returned to your inventory with PR link in lore
- 🎉 Level up sound + happy villager particles

**Failure:**
- ❌ Chat message: "Failed to implement ticket: [error]"
- 📖 Book returned with error details
- 🚫 Villager "no" sound

### 3.5 Check the PR

Click the PR link in chat or check GitHub:
```
https://github.com/bynder/byndiecraft-plugin/pulls
```

You should see a PR titled:
```
TEST-123: Add health check endpoint to API
```

With body:
```
## 🎮 Implemented by Byndiecraft AI System

**Jira Ticket:** https://bynder.atlassian.net/browse/TEST-123
**Requested by:** YourMinecraftName (in Minecraft)

This PR was automatically generated by throwing a book into the AI Hopper in Minecraft! 🤖
```

## Step 4: Demo Flow for Hackathon

### The Story

1. **Show the Jira board** with tickets in item frames
2. **Grab a ticket book** from the "To Do" column
3. **Walk to the AI Lab** (dramatic building with the hopper)
4. **Throw the book** into the glowing hopper
5. **Watch the magic** - particles, sounds, chat messages
6. **Show the MCP server logs** in a terminal (for audience)
7. **Wait ~15 seconds** while Claude works
8. **Book returns** with PR link
9. **Click the PR link** - browser opens to GitHub
10. **Show the PR** - code changes, commit message, everything automated
11. **Walk back to board** and move book to "In Review" column
12. **Jira updates** automatically

### Tips for the Demo

- **Pre-create some tickets** in Jira with clear, simple requirements
- **Test the flow** before the demo
- **Have a backup** in case of network issues
- **Show the MCP server logs** on a second screen for the "wow factor"
- **Emphasize**: "This is all happening from inside Minecraft!"

## Troubleshooting

### MCP Server Issues

**"Cannot find module 'express'"**
```bash
cd mcp-server
npm install
```

**"gh: command not found"**
```bash
# Mac
brew install gh

# Linux
# See: https://cli.github.com/
```

**"AWS credentials not found"**
```bash
aws configure --profile development
```

### Minecraft Plugin Issues

**"AI system disabled"**

Check `config.yml`:
```yaml
ai:
  enabled: true
```

Restart the server.

**"AI Hopper: Not configured"**

Run in Minecraft:
```
/jiraboard setaihopper
```

While looking at a hopper.

**"No player found near hopper"**

Make sure you're within 10 blocks of the hopper when throwing the book.

**"MCP server error: ECONNREFUSED"**

The MCP server is not running. Start it:
```bash
cd mcp-server
npm start
```

### GitHub Issues

**"gh: not authenticated"**
```bash
gh auth login
```

**"Permission denied"**

Make sure you have push access to the repository.
